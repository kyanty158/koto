# KOTO 要件定義書 (SwiftUIネイティブ, rev 3)

> 現行の SwiftUI + Core Data 実装に準拠して再定義。起動から入力までの速さと、ローカル通知による「思い出し」を最優先する。

---

## 0. 用語
- KOTO: 本アプリ
- Write: 「書く」タブ
- View: 「見る」タブ
- Reminder: リマインダー（ローカル通知）

---

## 1. 目的・非目的
### 1.1 目的
- 起動→入力フォーカスまで最小ステップ（自動フォーカス、キーボード即表示）
- 保存も破棄もワンタップで完結（ドラッグ依存を排除）
- オフラインでも安全・高速に保存し、確実にローカル通知を鳴らす

### 1.2 非目的（現行リリース）
- 同期/共有/共同編集
- サインインやクラウド依存のワークフロー
- 複雑なタグ管理 UI（`#tag` の自動抽出のみ）

---

## 2. 対象ユーザー
- 「思いついた瞬間に一行書きたい」ライトユーザー
- 移動中・片手操作が多く、通知でリマインドしたい人

---

## 3. 提供価値
- 起動直後に書き始められる（フォーカス済み）
- ボタン3つで「破棄 / ただ保存 / 保存＋通知」が完結
- ローカル完結のため通信不安定でも動く
- 通知アクションから編集に即復帰可能

---

## 4. コアUX（実装準拠）
### 4.1 書く（Write）
- 起動後すぐ TextEditor にフォーカス
- アクションボタン:
  - 破棄（現在のテキストをクリア）
  - ただ保存（通知なしで保存）
  - 保存＋通知（プリセット一覧シートから選択して保存・通知予約）
- リマインダープリセット: 10/20/30/45/60/90/120/180/240/360 分後、今夜(20:00)、明日朝(9:00)
- 2本指タップで直前の保存/破棄を Undo
- 保存時に `#tag` を自動抽出して小文字・重複排除
- Haptic フィードバックあり

### 4.2 見る（View）
- 「これからのリマインド」最大6件を先頭に固定表示（期限順）
- 履歴は作成日時降順
- 全文および `#tag` 検索（検索バー）
- 選択モード: 全選択/解除、まとめて削除
- 行のコンテキストメニュー: 完了トグル、リマインダー解除、削除
- タップで編集シートを開く

### 4.3 編集（Edit）
- 本文・完了フラグ・リマインダーを編集
- 保存で Core Data 更新 + 通知再スケジュール
- 削除で DB から削除し通知もキャンセル

### 4.4 通知
- ローカル通知 (UNUserNotificationCenter)
- アクション: 完了 / スヌーズ(+15m) / 編集を開く
- 通知から編集要求を受け取り、アプリ側で対象メモを開く

### 4.5 広告（テストモードのみ UI 露出）
- Google Mobile Ads SPM 依存
- テスト ID でバナー常時表示（上・下）＋テスト用 Ads タブ
- 本番ビルドでは環境変数 `KOTO_TEST_MODE` を見て切替可能（実装済み）

---

## 5. データモデル（Core Data: MemoEntity）
- id: Int64（AutoIncrementer）
- text: String
- createdAt / updatedAt: Date
- reminderAt: Date?
- isDone: Bool
- inlineTags: [String]（`#tag` を抽出・小文字化）
- ユニークキー: id

---

## 6. 非機能要件（目標）
- Cold→入力可能: ≤1.2s(P50)/≤1.6s(P90)
- Warm→入力可能: ≤0.8s(P50)/≤1.2s(P90)
- 通知・保存エラー時はトーストで明示
- オフラインでも保存・通知予約が成立

---

## 7. プラットフォーム/技術
- iOS 15+ / SwiftUI
- データ: Core Data（動的に NSManagedObjectModel を構築、.xcdatamodeld 不要）
- 依存: GoogleMobileAds (SPM) ※現状テスト ID 固定
- 同期・Firebase・Analytics は未使用（将来拡張）

## 10. 受け入れ条件（AC）
- AC-01: 起動でキーボードが自動表示される
- AC-02: 「保存＋通知」でプリセット選択 → 保存 & 通知予約される
- AC-03: 「破棄」で入力をクリアでき、Undo で戻せる
- AC-04: 「ただ保存」で通知なし保存ができる
- AC-05: 通知アクション（完了/スヌーズ/編集）が反映される
- AC-06: 検索バーで全文/`#tag` 検索ができる
- AC-07: 選択モードで複数削除ができる

---

## 11. テスト計画（現行）
- 単体: タグ抽出、リマインダープリセットの日時解決、Undo の保存/破棄復元
- 統合: 通知アクション → Core Data 反映 → UI 反映
- 実機: iOS 最新/1世代前で冷/温起動計測、通知動作、広告表示（テスト ID）

---
